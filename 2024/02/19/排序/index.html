数据结构-排序

排序

定义

记录以顺序表存储

插入排序

基本思想：

每步将一个待排序对象，按其关键码大小，插入到前面已经排好序的一组对象的适当位置上，直到对象全部插入为止（即边插入边排序，保证子序列中随时都是排好序的）

分类：

直接插入排序

  • 步骤：

    :one:在R[1…i-1]中查找R[i]的插入位置，R[1…j].key <= R[i].key < R[j+1…i-1].key

    :two:将R[j+1…i-1]中的所有记录均后移一个位置

    :three:将R[i] 插入到R[j+1]的位置上

  • 算法：

  • 算法分析：

    • 在平均情况下，直接插入排序关键字的比较次数和记录移动次数均约为：n^2^ /4

    • 时间复杂度为：O(n^2^ )

    • 空间复杂度为：O(1) <·····只需要一个记录的辅助空间r[0]

    • 是一种稳定的排序方法

    • 也适用于链式存储结构，只是在单链表上无需移动记录，只需修改相应的指针

    • 更适合于**初始记录基本有序（正序）**的情况，当初始记录无序，n较大时，此算法时间复杂度较高，不宜采用

    • 所以只能用于顺序结构，不能用于链式结构

    • 适合初始记录无序、n较大时的情况

折半插入排序
  • 算法：
  • 算法分析：
    • 时间复杂度为：O(n^2^ )
    • 空间复杂度为：O(1)
    • 是一种稳定的排序方法
    • 当n较大时，总关键码比较次数比直接插入排序的最坏情况要好得多，但比其最好情况要差（总关键码比较次数：直接插入最好情况 < 折半 << 直接插入最坏情况）
    • 在对象的初始排列已经按关键码排好序或接近有序时，直接插入排序比折半插入排 序执行的关键码比较次数要少
    • 折半插入排序的对象移动次数与直接插入排序相同，依赖于对象的初始排列
    • 减少了比较次数，但没有减少移动次数
    • 平均性能优于直接插入排序
希尔排序

  • 基本思想：

    先将整个待排记录序列分割成若干子序列，分别进行直接插入排序，待整个序列中的记录“基本有序” 时，再对全体记录进行一次直接插入排序

    • 子序列的构成不是简单地”逐段分割”。而是，将相隔某个增量dk的记录组成一个子序列让增量dk逐趟缩短（例如依次取5,3,1）直到dk＝1为止
    • 小元素跳跃式前移最后一趟增量为1时， 序列已基本有序平均性能优于直接插入排序

  • 算法：

  • :exclamation:注意：关于“一趟”：[0,0+dk)共dk组，每组都完成一次排序为“一趟”

    ​ r[0]只是暂存单元，不是哨兵，当j<=0时，插入位置巳找到

  • 算法分析：

    • 时间复杂度：未有定论。有人在大量的实验基础上推出：当 n 在某个特定范围内，希尔排序所需的比较和移动次数约为 **n^1/3^**当 n—>∞ 时，可减少到 n(log~2~n)^2^
    • 空间复杂度：O(1)<·····只需要一个辅助空间r[0]
    • 记录跳跃式地移动导致排序方法是不稳定的
    • 只能用于顺序结构，不能用于链式结构
    • 增量序列可以有各种取法，但应该使增量序列中的值没有除1之外的公因子，并且最后一个增量值必须等于1
    • 适合初始记录无序、n较大时的情况
总结：
直接插入排序 折半插入排序 希尔排序
时间复杂度 O(n^2^ ) O(n^2^ ) 未定
空间复杂度 O(1) O(1) O(1)
稳定性 稳定 稳定 不稳定
使用场景 初始记录无序、n较大 n较大，初始排列已经按关键码排好序或接近有序 始记录无序、n较大

两两比较，如果发生逆序则交换，直到所有记录都排好序为止

快速排序

  • 基本思想：

    通过一趟排序，将待排序记录分割成独立的两部分，其中一部分记录的关键字均比另一部分记录的关键字小，则可分别对这两部分记录进行排序，以达到整个序列有序

    :one:任取一个元素 (如第一个) 为中心

    :two:所有比它小的元素一律前放，比它大的元素一律后放，形成左右两个子表

    :three:对各子表重新选择中心元素并依此规则调整，直到每个子表的元素只剩一个

  • 算法：

  • 算法分析：

    • 平均计算时间是：O(nlog~2~n)
    • 实验结果表明： 就平均计算时间而言，快速排序是我们所讨论的所有内排序方法中最好的一个 :+1:
    • 快速排序是递归的，需要有 一个栈存放每层递归调用时参数（新的 low和high）
    • 最大递归调用层次数与递归树的深度一致， 因此，要求存储开销为：O(log~2~n)
    • 稳定性：不稳定，可选 任一元素为枢轴
    • 分情况讨论：
      • 最好：划分后，左侧右侧子序列的长度相同，空间效率：O(log~2~n)
      • 最坏：初始记录序列按关键字有序或基本有序，快速排序将蜕化为起泡排序。递归树成为单支树，每次划分只得到一个比上一次少一个 对象的子序列，必须经过 n-1 趟才能把所有对象定位，而且第 i 趟需要经过 n-i 次关键码比较才能找到第 i 个对象的安放位置。此时，栈的最大深度为n。O(n)。

  • :exclamation:注意：“一趟”指: 对一个（子）表完成排序，即对于一个元素，在比较区域内，左边均小，右边均大

冒泡排序 快速排序
时间复杂度 O(n^2^) O(nlog~2~n)
空间复杂度 O(1) O(n) ~ O(log~2~n)

堆排序：

  • 定义：

    • 堆排序 (Heap Sort) 是一种树形选择排序，在排序过程中，将待排序的记录 r[l…n]看成是一棵完全二叉树的顺序存储结构，利用完全二叉树中双亲结点和孩子结点之间的内在关系，在当前无序的序列中选择关键字最大（或最小）的记录

    • 若将和此序列对应的一维数组（ 即以一维数组做此序列的存储结构） 看成是一个完全二叉树， 则堆实质上是满足如下性质的完全二叉树：树中所有非终端结点的值均不大于（或不小于）其左、右孩子结点的值(根结点最大或最小)

    • n个元素的序列{k~1~,k~2~,…，k~n~}称之为堆， 当且仅当满足以下条件时：

      {ki≤k2iki≤k2i+1或{ki≥k2iki≥k2i+1\begin{cases}k_i\leq k_{2i}\\k_i\leq k_{2i+1}\end{cases}或\begin{cases}k_i\geq k_{2i}\\k_i\geq k_{2i+1}\end{cases} {ki​≤k2i​ki​≤k2i+1​​或{ki​≥k2i​ki​≥k2i+1​​

  • 分类：大顶堆/小顶堆；大根堆/小根堆

  • 性质：

    • 堆是一棵采用顺序存储结构的完全二叉树
    • 堆的根结点是关键字序列中的最小(或最大)值，分别称为小(或大)根堆
    • 从根结点到每一叶子结点路径上的元素组成的序列都是按元素值(或关键字值)非递减(或非递增)的
    • 堆中的任一子树也是堆

  • 堆排序：

    利用堆顶记录的关键字值最小(或最大)的性质，从 当前待排序的记录中依次选取关键字最小(或最大)的记录，就可以实现对数据记录的排序，这种排序方法称为堆排序

  • 基本思想：

    :one:将无序序列建成一个堆

    :two:输出堆顶的最小（大）值

    :three:使剩余的n-1个元素又调整成一个堆，则可得到 n个元素的次小值

    :four:重复执行，得到一个有序序列

  • 算法：

    :one:堆调整算法

    :two:堆排序

  • 算法分析：

    • 时间效率：O(nlog~2~n)
    • 空间效率：O(1)
    • 稳定性：不稳定
    • 适用于n较大的情况
    • 只能用顺序存储实现，不能用链式存储
    • 堆排序的时间主要消耗在建初始堆和调整建立新堆时进行的反复“筛选”上。堆排序在最坏情况下，其时间复杂度也为O(nlog2n)，这是堆排序的最大优点。无论待排序列中的记录是正序还是逆序，都不会使堆排序处于“最好”或“ 最坏”的状态

归并排序

定义

  • 归并：将两个或两个以上的有序表组合成一个新有序表

  • 2-路归并排序：子序列总是两两归并

    :one:初始序列看成n个有序子序列，每个子序列长度为1

    :two:两两合并，得到 n/2 个长度为2或1的有序子序列

    :three:再两两合并，重复直至得到一个长度为n的有序序列为止

算法：

算法分析：

  • 时间效率：O(nlog~2~n)
  • 空间效率：O(n)<·····需要一个与原始序列同样大小的辅助序列
  • 稳定性：稳定
  • 可用于链式结构，且不要附加存储空间，但递归实现时仍需要开辟相应的递归工作栈。
  • 整个归并排序仅需 log~2~n 趟
  • 两个顺序表合并的时间复杂度为O(m+n)，m和n分别为顺序表表长
