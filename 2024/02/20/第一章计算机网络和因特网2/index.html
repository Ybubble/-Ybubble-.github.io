第一章-计算机网络和因特网（一）

1.4分组交换网络中的时延、丢包和吞吐量

1.4.1分组交换网中的时延概述

  • 节点总时延（total node delay）=节点处理时延（nodal processing delay）+排队时延（queuing delay）+传输时延（transmission delay）+传播时延（propagation ddelay）

    • 仅当在该链路没有其他分组正在传输并且没有其他分组排在该队列前面时，才能在这条链路上传输该分组
    • 如果该链路当前正繁忙或没有其他分组已经在该链路上排队，则新到达的分组将参与排队

  • 处理时延，d~proc~: nodal processing delay

    • 检查分组首部、决定将该分组导向何处所需要的时间
    • 检查比特级别的差错（该差错出现在上游结点向该路由器传输这些分组比特的过程中）check bit errors
    • determine output link
    • 微秒或更低的数量级

  • 排队时延，d~queue~：queueing delay

    • 在队列中，当分组在链路上等待传输时。time waiting at output link for transmission
    • 一个特定分组的排队时延长度取决于：先期到达的正在排队等待向链路传输的分组数量。depends on congestion level of router
    • 毫秒到微秒量级

  • 传输时延，d~trans~: transmission delay

    • L/R（将所有分组的比特推（传输）向链路所需要的时间。the amount of time required to transmit (push) all of the packet’s bits into the link），L：比特，分组长度，R：，bps（b/s），两个路由器之间 的链路传输速率）
    • 毫秒到微秒量级

  • 传播时延，d~prop~: propagation delay

    • the time required to propagate from the beginning of the link to next router

    • 取决于物理链路，范围2×10^8^ ~ 3×10^8^m/s，等于或略小于光速

    • 传播时延=d/s（d：两个路由器之间的距离，s：链路的传播速率）

    • 在广域网中，传播时延为毫秒量级

结点处理时延（d~proc~) 排队时延(d~queue~) 传输时延(d~trans~) 传播时延(d~prop~)
量级 微秒or< 毫秒到微秒 毫秒到微秒
microsecond to millisecond		 毫秒
备注 对于由同步卫星链路互联的两台路由器来说是几百毫秒 对于10Mbps和更高的传输速率（如对于LAN）而言是微不足道的
对于通过低速拨号调制解调器链路发送的长因特网分组而言，可能是数百毫秒
  • 处理时延通常是微不足道的，但它对一台路由器的最大吞吐量有重要影响。最大吞吐量：一台路由器能够转发分组的最大速率

1.4.2排队时延和丢包

  • 衡量排队时延：流量到达该队列的速率、链路的传输速率、到达流量的性质（流量是周期性到达还是以突发形式到达）

  • 流量强度（tranffic intensity）=La/R

    • L：分组由L比特组成，packet length (bits)；a：分组到达队列的平均速率，以bps即b/s为单位，average packet arrival rate (packets/s)；R：传输速率，transmission rate (bps)；La：比特到达队列的平均速率

    • La/R>1：比特到达队列的平均速率超过从该队列传输出去的速率，排队时延趋于无穷大，La/R > 1: more “work” arriving than can be serviced, average delay infinite

    • La/R≤1：

      • 到达流量的性质影响排队时延
      • 如果分组周期性到达（每L/R秒到达一个分组），则每个分组将到达一个空队列中，不会有排队时延
      • 如果分组以突发形式到达，则有很大的突发时延
        • 假若每（L/R）N秒同时到达N个分组，则传输的第一个分组没有排队时延，传输的第二个分组就有L/R秒的排队时延
        • 一般地，din个传输的分组有（n-1）L/R秒的排队时延

    • La/R->0：平均排队时延接近0

    • La/R->1：存在到达率超过传输能力的时间间隔，这时将形成队列，queueing delay large

      • 随着流量强度接近1，平均排队时延迅速增加

      • 高速路一场车祸—》全段堵车

  • 丢包

    • 排队容量有限，随着流量强度接近1，排队时延并不实际取项无穷大
    • 当到达的分组发现一个满的队列，路由器将丢弃（drop）该分组，即该分组将会丢失（lost）
    • lost packet may be retransmitted by previous node, by source end system, or not at all

  • 衡量一个结点的性能：时延、分组丢失的概率

1.4.3端到端时延

  • 假设源主机和目的主机之间有N-1台路由器，无拥塞（此时排队时延微不足道）：

    d~end-end~=N(d~proc~+d~trans~+d~prop~)

1.4.4计算机网络中的吞吐量

  • Throughput(吞吐量): rate (bits/time unit) at which bits transferred between sender/receiver

  • 计算机网络的性能衡量：时延、丢包、端到端吞吐量

  • 瞬时吞吐量（instantaneous throughput）：任一瞬间主机接收另一主机发送的文件的速率（bps），rate at given point in time

  • 平均吞吐量（average throughput）=F/T （bps，F的单位是比特），rate over longer period of time

1.5协议层次及其服务类型

1.5.1分层的体系结构

  • 每个层次提供服务的方式：

    • 在该层中执行了某些动作，performing its own internal-layer(内层) actions
    • 使用直接下层的服务，relying on services provided by layer below

  • 协议分层

    • 优：
      • 提供结构化方式来讨论系统组件
      • explicit structure allows identification, relationship of complex system’s pieces
      • modularization eases maintenance, updating of system 降低维护和更新成本
    • 缺：
      • 一层可能冗余较低层的功能。例如，许多协议栈在基于每段链路和基于端到端两种情况下，都提供了差错恢复，one layer may duplicate lower-layer functionality
      • 某层的功能可能需要仅在其他层才出现的信息（如时间戳值），functionality at one layer may need information (e.g. timestamp value) that is present only in another layer
    • 各层的所有协议被称为协议栈（protocol stack）

  • 五层因特网协议栈

    名称 分组的名称
    应用层 message
    运输层 报文段segment
    网络层 数据段datagram
    链路层 帧frame
    物理层 帧frame
    • 应用层：application：supporting network applications
      • 网络应用程序和它们的应用层协议留存的地方
      • HTTP：提供web文档的请求和传送
      • SMTP：提供电子邮件报文的传输
      • FTP：提供两个端系统之间的文件传送
      • 应用层协议分布在多个端系统上
      • 一个端系统中的应用程序使用协议与另一个端系统中的应用程序交换信息的分组
      • 位于应用层的信息分组称为报文（message），this packet of information at the application layer as a message（消息/报文）
      • always implemented in software in the end systems
    • 运输层，transport：process-process data transfer
      • 运输层在应用程序端点之间传送应用层报文
      • 协议：TCP、UDP，利用任意一个都能运输应用层报文
        • TCP：
          • 向应用程序提供面向连接的服务，包括了应用层报文向目的地的确保传递和流量控制（即发送方/接收方速率匹配）
          • 将长报文划分为短报文
          • 并提供拥塞控制机制：当网络拥塞时，原抑制其传输速率
        • UDP：
          • 向应用程序提供无连接服务
          • 一种不提供不必要服务的服务，没有可靠性、没有流量控制、没有拥塞控制
      • 运输层分组称为报文段（segment）
    • 网络层，network：routing of datagrams from source to destination
      • 负责将网络层分组从一台主机移动到另一台主机
      • often a mixed implementation of hardware and software
      • 分组名称：数据报（datagram）
      • 一台源主机中的因特网运输层协议（TCP或UDP）向网络层递交运输层报文段和目的地址
      • 协议：
        • IP协议：
          • 定义了数据报中各个字段
          • 定义了端系统和路由器如何运用于这些字段
        • 路由选择协议
          • 决定路由
          • 使数据报根据该路由从原传输到目的地
    • 链路层，link: data transfer between neighboring network elements
      • 目的：将分组从一个结点（主机或路由器）移动到路径的下一个结点
      • typically implemented in a network interface网络接口
      • 优链路层提供的服务取决于应用于该链路的特定链路层协议
      • 协议：
        • 如以太网、WiFi和电缆接入网的DOCSIS协议， Ethernet, 802.11(Wi-Fi), PPP(点对点协议)
        • 一个数据报可能被沿途不同链路上的不同链路层协议处理
      • 分组名称：帧（frame）
    • 物理层
      • 链路层的任务：将整个帧从一个网络元素移动到邻近的网络元素
      • 物理层的任务：将该帧中的一个一个比特从一个结点移动到下一个结点
      • typically implemented in a network interface
      • 协议：与链路相关，并且进一步与该链路的实际传输媒体相关

  • OSI模型

    • 国际标准化组织（ISO，International Organization for Standardization）提出计算机网络组织大约7层，称为开放系统互联（OSI，Open System Interconnect）模型

    • 七层ISO（少用）

    • 表示层，presentation：

      • 使通信的应用程序能够解释交换数据的含义，allow applications to interpret meaning of data
      • 服务包括数据压缩compression、数据加密encryption、护具描述、machine-specific conventions（特定机器的约定）

    • 会话层，session：

      • 提供数据交换定界和同步功能，包括建立检查点和恢复方案的方法，synchronization, check-pointing, recovery of data exchange
    • 应用层

1.5.2封装（encapsulation）

  • M：应用层报文（application-layer message）
  • M+H~t~（运输层首部信息）=运输层报文段（transport-layer segment）==》封装了应用层报文
    • 附加信息可能有
      • 运行接收端运输层向上向适当的应用程序交付报文的信息
      • 差错检测位信息：让接收方能够判断报文中的比特是否在途中已经改变
  • +H~n~（网络层首部信息）=网络层数据报（network-layer datagram）
  • =》链路层帧（link-layer frame）
  • 每一层，一个分组有两种类型的字段：首部字段+有效载荷字段（payload field）
    • 有效载荷通常来自上一层的分组

1.6面对攻击的网络

malware：恶意软件

botnet：僵尸网络

  • malware(恶意软件) can get in host from:

    • virus (病毒):
      • self-replicating infection by receiving/executing object (e.g., e-mail attachment)
      • 一种需要某种形式的用户交互来感染用户设备的恶意软件
    • worm (蠕虫):
      • self-replicating infection by passively receiving object that gets itself executed (e.g. 系统漏洞)
      • 一种无需任何明显用户交互就能进入设备的恶意软件

  • spyware malware(间谍类恶意软件) can record keystrokes (键盘敲击), web sites visited, upload info to collection site

  • infected host can be enrolled in botnet (僵尸网络), used for spam. DDoS attacks

  • Denial of Service attack(DoS拒绝服务攻击) :

    • attackers make resources (server, bandwidth) unavailable to legitimate traffic (合法流量) by overwhelming (淹没) resource with bogus traffic (伪造的流量)

    • 1、select target

      2、break into (入侵) hosts around the network (see botnet)

      3、send packets to target from compromised hosts 受害主机

    • 类型

      • 弱点攻击
        • 向一台目标主机上运行的易受攻击的应用程序或操作系统发送制作精细的报文
        • 如果适当顺序的多个分组发送给一个易受攻击的应用程序或操作系统，该服务器可能停止运行，甚至崩溃
      • 带宽洪泛
        • 向目标主机发送大量的分组，数量之多使得目标的接入链路变得拥塞，使合法的分组无法到达服务器
      • 连接洪泛
        • 在目标主机中创建大量的半开或全开TCP连接，该主机因为这些伪造的连接而陷入困境，并停止接收合法的连接

  • 分布式DoS（Distribute DoS，DDoS）

    • 攻击者控制多个源并让每个源向目标猛烈发送流量

  • packet “sniffing嗅探” :

    • broadcast media (shared ethernet, wireless)
    • promiscuous (混杂) network interface reads/records all packets (e.g., including passwords!) passing by
    • 在无线传输设备的附近放置一台被动的接收机，该接收机就能得到传输的每个分组的副本
    • 记录每个流经的分组的副本的被动接收机称为分组嗅探器（packet sniffer）
    • 嗅探器也能部署在有线环境中

  • IP spoofing(IP欺骗): send packet with false source address
