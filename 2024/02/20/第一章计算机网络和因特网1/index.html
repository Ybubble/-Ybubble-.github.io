第一章-计算机网络和因特网（一）

1.1因特网

具体构成描述

  • 端系统通过通信链路(communication links)和分组交换机(packet switch)连接到一起

  • milions of connected computing devices：host=end system

  • many types of communication links made up of different types of physical media and radio spectrum

  • 物理媒体—>通信链路(communication links):

    • 同轴电缆、铜线(copper)、光纤(fiber)、无线电频谱(radio spectrum)、satellite(卫星)

    • 传输速率以比特/秒（bit/s,bps）来度量，transmission tate=bandwidth(带宽)

  • 分组交换机（packet switches）

    • 路由器、链路层交换机

    • 朝着最终目的地转发分组，forward packets（chunks of data）

    • 路由器(routers)：常用于网络核心

      链路层交换机(link-layer switches)：常用于接入网中

  • 路径：一个分组所经历的通信链路+交换机

  • 端系统通过因特网服务提供商接入因特网

  • ISP

    • 每个ISP是一个由多个分组交换机和多段通信链路组成的网络

  • 为端系统提供不同类型的网络接入：

    • 线缆调制解调器

      • （DLS？）住宅宽带接入

      • 高速局域网接入

      • 无线接入

      • 56kbps拨号调制解调器接入

  • 为内容提供者提供网络接入服务，将web站点直接接入因特网

  • 端系统互联—>ISP必须互联

    • 低层ISP通过国家的、国际的高层ISP（Level 3 Communications、AT&T、Sprint、NTT）互联

  • 高层ISP通过高速光纤链路互联的高速路由器组成

    • 每个ISP独立管理，运行着IP协议

  • 因特网标准协议（Internet standard）由因特网工程任务组（Internet Engineering Task Force，IETF）研发

  • IETF的标准文档为请求评论（Request For Comment，RFC），定义了用于web的TCP、IP、HTTP和用于电子邮件的SMTP等协议

服务描述

描述角度：为应用程序提供服务的基础设施

  • 应用程序（分布式应用程序，distributed application）：
    • 电子邮件、web冲浪、即使讯息、社交网络、因特网电视、IP语音（VoIP）、流式视频、分布式游戏、对等（peer-to-peer，P2P）、文件共享、因特网电视、远程注册等
    • 分布式—>涉及多台交换数据的端系统
    • 运行在端系统上，不运行在网络核心中的分组交换机中
  • communication infrastructure provided to apps
    • connection-oriented reliable service（面向连接可靠服务）：reliable data delivery from source to destination, called Transmission Control Protocol (TCP)
    • connectionless unreliable service ( 无连接不可靠服务): “best effort” (unreliable) data delivery, called User Datagram Protocol (UDP)
  • 分组交换机：促进端系统之间的数据传输，不关心作为数据的源或宿的应用程序
  • 应用程序接口（Application Programming Interface，API）：规定了运行在一个端系统上的软件请求因特网基础设施向运行在另一个端系统上的特定目的的软件交付数据的方式
    • Internet API is a set of rules that the sending program must follow
    • hooks that allow sending and receiving app programs to “connect” to Internet
    • provides service options, similar to postal service 类似邮局服务:
    • example: Alice send a letter to Bob
    • a). follow the rule
    • b). provide more than one service

协议

一个协议定义了在两个或多个通信实体之间交换的报文格式和次序，以及报文发送和/或接收一条报文或其他事件所采取的动作

protocols define format, order of msgs sent and received among network entities, and actions taken on msg transmission, receipt

  • entities change: machines rather than humans

  • all communication activity in Internet governed by protocols

  • 主机：

    • 客户client
    • 服务器server

  • 大部分提供搜索结果、电子邮件、web页面和视频的服务器都属于大型数据中心（data center）

网络边缘

  • network edge（网络边缘）

    • hosts: clients and servers 客户端和服务器
    • servers often in data centers (数据中心)

  • access networks，physical media (接入网络，物理介质): wired，wireless communication links

  • network core(网络核心):

    • interconnected routers
    • network of network

  • end systems (hosts主机):

    • at “edge of network”
    • run (host) application programs, e.g. Web，email

  • further divided client/server model ( 客户端 /服务器模型)

    • client host requests, receives service from always -on server

    • e.g. Web browser/server; email client/server

    • client: desktop PCs, PDAs, servers: powerful machines (informally)

  • peer -peer model (对等模型):

  • minimal (or no) use of dedicated servers

  • e.g. Skype，PPTV，Thunder

接入网

  • 接入网（access network）：将端系统连接到其边缘路由器（edge router）的物理链路
  • 边缘路由器：端系统到任何其他远程端系统的路径上的第一台路由器

:one:家庭接入（住宅接入网络，residential access nets）：DSL、电缆、FTTH、拨号、卫星

🔹DSL（digital subscribe line，数字用户线路）

  • 住户通常从（提供本地电话接入的）本地电话公司处获得DSL因特网接入

  • 当使用DSL时，本地电话公司就是它的ISP

  • 每个用户的DSL调制解调器使用现有的电话线（双绞铜线）与位于本地电话公司的**本地中心局（CO，central office）中的数字用户线接入复用器（DSLAM，DSL接入复用器）**来交换数据

  • 家庭的DSL调制解调器将得到的数据转换为高频音，以通过电话线传输给本地中心局

  • 来自多个家庭的模拟信号在DSLAM处转换回数字形式

  • 家庭电话线同时承载了数据和传统的电话信号，编码为不同的频率：

    • 高速下行信道，位于50kHz ~ 1MHz频段
    • 中速下行信道，位于4kHz ~ 50kHz频段
    • 普通的双向电话信道，位于0kHz ~ 4kHz频段

  • note：

    :one:Home Access (1):digital subscriber line (DSL，数字用户线路)

    • DSL Modem (DSL 调制解调器)：将数字信号调制到模拟载波信号上进行传输，并解调收到的模拟信号以得到数字信息的电子设备

    • Splitter (分离器)：separates data and telephone signals arriving to home and forwards data signal to DSL modem (from CO to Home)

    • DSLAM (digital subscriber line access multiplexer，数字用户线路接入复用器)：

      • analog signals（模拟信号） from many houses are translated back into digital format at the DSLAM
      • separates the data and phone signals and sends the data into the Internet（from Home to CO)

    • single DSL link appear three separate links

    • a telephone call and an Internet connection can share the DSL link at the same time

    • use existing telephone line to central office DSLAM

      • data over DSL phone line goes to Internet

      • voice over DSL phone line goes to telephone net

    • DSL classification

      • **ADSL (Asymmetric Digital Subscriber Line，非对称数字用户线) **, typically upstream 1Mbps， downstream 8Mbps

      • HDSL（High-speed Digital Subscriber Line,高速率数字用户线）， upstream and downstream typically 2Mbps

      • VDSL（Very High Speed Digital Subscriber Line，超高速数字用户线）, typically upstream 19.2Mbps，downstream 55Mbps

:small_blue_diamond:电缆因特网接入（cacble Internet access）

  • 混合光纤同轴（Hybrid Fiber Coax，HFC）网络

  • 电缆因特网接入需要特殊的调制解调器——电缆调制解调器（cable modem）

  • 电缆调制解调器通常是一个外部设备，通过一个以太网端口连接到家庭PC

  • 在电缆头端，**电缆调制解调器系统（Cable Modem Termination System）**起到如同DSL网络的DSLAM类似的功能：（来自许多下行家庭中的电缆调制解调器发送的）模拟信号—>数字信号

  • 电缆因特网接入的重大特征：共享广播媒体

    • 若几个用户同时经下行信道下载一个视频文件，则每个用户接收视频文件的实际速率远低于电缆总计的下行速率
    • 若很少的活跃用户进行web冲浪，则每个用户都可以以下行速率的全部速率接收web网页
    • 共享—>需要一个分布式多路访问协议来协调传输和避免碰撞

  • network of cable, fiber attaches homes to ISP router

:small_blue_diamond:光纤到户（Firber To The Home，FTTH），Home Access (3)

  • optical network terminator (ONT, 光网络终端，通常 称为“光猫”) similar to Modem
  • 光纤分布体系结构（竞争性），optical-distribution network architectures：
    • 主动光纤网络（Active Optical Network，AON，有源光网络）
    • 被动光纤网络（Passive Optical Network，PON，无源光网络）
  • Each home has ONT (光猫)，users connect a home router to the ONT to access Internet (国内大多是路由器和光猫的一体机)
  • OLT providing conversion between optical and electrical signals
  • Splitter combines a number of homes (typically <100) onto a single, shared optical fiber（光纤） (from home to OLT)
  • PON
    • 每个家庭有一个光纤网络端接服务器（Optical Network Terminator，ONT），由专门的光纤连接到邻近的分配器（splitter）
    • 光纤线路端接器（Optical Line Terminator，OLT）提供光信号和电信号之间的转换，经本地电话公司的路由器与因特网相连
    • 所有从OLT发送到splitter的分组在splitter（类似于一个电缆头端）处复制

物理媒体（physical medium）

技术 物理媒体

  • 分类

    • 引导型媒体（guided media）
      • 电波沿着固体媒体前行
      • 光缆、双绞铜线、同轴电缆
    • 非引导型媒体（unguided media）
      • 电波在空气或外层空间中传播
      • 无线局域网、数字卫星频道

  • 双绞铜线

    • 最便宜、最普遍，常用于电话网，少用于电话线

    • 一对电线构成一个通信链路

    • 无屏蔽双绞线（UTP，Unshielded Twisted Pair）：

      • 常用在建筑物内的计算机网络中，即局域网（LAN）中

      • 数据传输速率：10Mbps~10Gbps，取决于线的粗细、距离。The data rates can be achieved depend on the thickness of the wire and the distance between transmitter and receiver

    • 高速LAN联网的主要方式，也常用于住宅因特网接入

    • 双绞线按屏蔽层有无可分为

    • UTP: Unshielded Twisted Pair (非屏蔽双绞线)

    • STP：Shielded Twisted-Pair （屏蔽双绞线），在芯线外增加铝箔层减少信号衰减

    • (2) 双绞线按照电气性能（频率、信噪比）可分为

      • 一类线（CAT1）：线缆最高频率带宽是750kHZ，用于报警系统，或只适用于语音传输（一类标准主要用于八十年代 初之前的电话线缆），不同于数据传输。
      • 二类线（CAT2）：线缆最高频率带宽是1MHZ，用于语音传输和最高传输速率4Mbps的数据传输，常见于使用4Mbps规范令牌传递协议的旧的令牌网。
      • 三类线（CAT3）：该电缆的传输频率16MHz，最高传输速率为10Mbps，主要应用于语音、10Mbit/s以太网 （10BASE-T）和4Mbit/s令牌环，最大网段长度为100m，采用RJ形式的连接器，目前已淡出市场（早期电话线）。
      • 四类线（CAT4）：该类电缆的传输频率为20MHz，用于语音传输和最高传输速率16Mbps（指的是16Mbit/s令牌环）的数据传输，主要用于基于令牌的局域网和10BASE-T/100BASE-T。最大网段长为100m，采用RJ形式的连接器，未被广泛采用。
      • 五类线（CAT5）：该类电缆增加了绕线密度，外套一种高质量的绝缘材料，线缆最高频率带宽为100MHz，最高传输率为100Mbps，用于语音传输和最高传输速率为100Mbps的数据传输，主要用于100BASE-T和1000BASE-T网络，最大网段长为100m，采用RJ形式的连接器。这是最常用的以太网电缆。
      • 超五类线（CAT5e）：超5类具有衰减小，串扰少，并且具有更高的衰减与串扰的比值（ACR）和信噪比（Structural Return Loss）、更小的时延误差，性能得到很大提高。超5类线主要用于千兆以太网（1000Mbps=1Gbps）。
      • 六类线（CAT6）：该类电缆的为250MHz，六类布线系统在 200MHz时综合衰减串扰比（PS-ACR）应该有较大的余量，它提供2倍于超五类的带宽(2Gbps)。六类布线的传输性能远远高于超五类标准，最适用于传输速率高于1Gbps的应用。要求的布线距离为：永久链路的长度不能超过90m，信道长度不能超过100m。CAT6A标准的传输速率可达到10Gbps，使用在万兆以太网中。
      • 七类线（CAT7）：传输频率为600MHz，可用于10Gbps(万兆)以太网。

1.3网络核心(The Network Core)

  • 网络核心：由互联因特网端系统的分组交换机和链路构成的网状网络，mesh of interconnected routers and links that interconnects routers

    • interconnected routers
    • network of networks

  • two fundamental approaches to moving data: circuit switching（电路交换） and packet switching（包交换）

    • circuit-switched networks: the resources needed along a path for communication between the end systems are reserved

    • packet-switched networks: these resources are not reserved; a session’s messages use the resources on demand

1.3.1分组交换（包交换，packet switching）

  • packet-switching: hosts break application-layer messages into packets (分组/数据包)

    • forward packets from one router to the next，across links on path from source to destination
    • each packet travels through communication links and packet switches (e.g. routers，link-layer switches)
    • each packet transmitted at full link capacity

  • 报文（message）：

    • 在各种网络应用中，端系统彼此交换报文
    • 可以执行一种控制功能、包含数据
    • 源将报文划分为较小的数据块——分组（packet）
    • 每个分组都通过通信链路和分组交换机（packet switch）（交换机主要有两类：路由器和链路层交换机）传送
    • 分组以等于该链路最大传输速率的速度传输通过通信链路
    • 如果某源端系统或分组交换机经过一条链路发送一个L比特的分组，链路的传输速率为R比特/秒，则传输该分组的时间为L/R秒

  • 存储转发传输（store-and-forward transmission）

    • 分组交换机在链路的输入端使用存储转发传输机制
    • 在交换机能够开始向输出链路传输该分组的第一个比特之前，必须接收到整个分组
    • 由N条速率均为R的链路组成的路径（所以，在源和目的地之间有N-1台路由器），端到端的时延：d=N（$$\frac{L}{R}$$）
    • Most packet switches use store and-forward transmission

  • 排队时延和分组丢失

    • 每个分组交换机有多条链路与之相连
    • 对于每条链路，该分组交换机有一个输出缓存（output buffer）（也称为输出队列（output queue），用于存储路由器准备发往那条链路的分组（存储转发时延）
    • each packet switch has multiple links, each link has an output buffer
    • 排队时延（queue delay）：如果到达的分组需要传输到某条链路，但该链路正忙于传输其他分组，则该到达分组必须在该输出缓存中等待，variable and depend on level of congestion in network
    • 分组丢失（丢包，packet lost）：一个分组到达时，若缓存已被其他等待传输的分组完全充满了，则到达的分组或已经排队的分组之一将被丢弃

  • 转发表（forwarding table）和路由选择协议（routing protocol）

    • 分组的首部包含有目的地的IP地址，当一个分组到达网络中的路由器时，路由器检查该分组的目的地址的一部分
    • routing: determines source destination route taken by packets
    • forwarding: move packets from router’s input to appropriate router output
    • 每台路由器有一个转发表（forwarding table），用于将目的地址（或目的地址的一部分）映射为输出链路
    • 路由选择协议（routing protocol）：用于自动地设置转发表
1.3.2电路交换（circuit switching）
  • 通过网络链路和交换机移动数据的两种基本方法：电路交换、分组交换
  • 在电路交换网络中，在端系统间通信会话器件，预留了端系统间通信沿路所需要的资源（缓存、链路传输速率）。在分组交换网络中，这些资源不是预留的，会话的报文按需使用这些资源，其后果是不得不等待（排队）接入通信线路
  • Circuit switching is commonly used in traditional telephone networks (传统电话网络)
  • 电路交换中的复用
    • 链路中的电路是通过**频分复用（FDM，Frequency-Division Multiplexing）或时分复用（TDM，Time-Division Multiplexing）**来实现的
    • FDM：
      • 链路的频谱由跨越链路创建的所有连接所共享
      • 在电话网络中，这个频段通常具有4kHz的宽度（即4000赫兹或每秒4000周），该频段的宽度称为带宽（bandwidth）
      • 调频无线电台也使用FDM来共享88~108MHz的频谱，每个电台被分配一个特定的频段
      • the link dedicates a frequency band (频段) to each connection for the duration（持续时间） of the connection
      • the width of the band is called the bandwidth (带宽).
      • each circuit continuously gets a fraction of the bandwidth
    • TDM：
      • 时间被划分为固定区间的帧，每帧又被划分为固定数量的时隙
      • 当网络跨越一条链路创建一条连接时，网络在每个帧中为该连接指定一个时隙。这些时隙专门由该连接单独使用，一个时隙（在每个帧内）可用于传输该连接的数据
      • time is divided into frames(帧) of fixed duration, and each frame is divided into a fixed number of time slots (时间片)
      • network dedicates one time slot in every frame to this connection, i.e. with one time slot available for use (in every frame) to transmit the connection’s data
      • each circuit gets all of the bandwidth periodically during brief intervals of time (slots)
分组交换与电路交换的对比
  • 分组交换
    • 不适合实时服务：端到端时延是可变的和不可预测的（主要因为排队时延的变动和不可预测所致）
    • 提供了比电路交换更好的宽带共享
    • 比电路交换更简单、更有效、实现成本更低
    • Packet switching (包交换): these resources are not reserved, use the resource on demand
    • Disadvantage: excessive congestion possible
      • packet delay and loss
      • protocols needed for reliable data transfer, congestion control
  • circuit switching (电路交换)
    • network must establish a connection between before send the information: the connection is called a circuit(电路).
    • the network establishes the circuit, it also reserves a constant transmission rate in the network’s links
    • transfer the data to the receiver at the guaranteed constant rate
Packet switching versus circuit switching
  • Critics of packet switching (批评者):
    • packet switching is not suitable for real-time services (for example, telephone calls and video conference calls) because of its variable and unpredictable end-to-end delays
  • Proponents of packet switching (支持者):
    • it offers better sharing of transmission capacity than circuit switching
    • is simpler, more efficient，and less costly to implement than circuit switching
      • more efficient: packet switching allows more users to use network
  • Packet switching versus circuit switching
    • the trend has certainly been in the direction of packet switching, e.g. expensive overseas telephone call
    • great for burst data (突发数据)
      • resource sharing
      • simpler, no call setup
    • Disadvantage: excessive congestion possible
      • packet delay and loss
      • protocols needed for reliable data transfer, congestion control
    • Q: How to provide circuit-like behavior?
      • bandwidth guarantees needed for audio/video apps
      • still an unsolved problem (chapter 7)
1.3.3网络的网络

  • ISP互联=》创建网络的网络

  • End systems connect to Internet via access ISPs

    • don’t have to be telco or cable company
    • can be residential，company and university

  • Access ISPs in turn must be interconnected (互相连接).

    • So that any two hosts can send packets to each other

  • Resulting network of networks is very complex

    • Evolution was driven by economics and national policies rather than by performance considerations

  • 互联接入ISP的目的：使所有端系统能够彼此发送分组

  • 网络结构1：用单一的全球承载ISP互联接入所有ISP， network of networks-single global ISP

  • 网络结构2：由数十万接入ISP和多个全球承载ISP组成，network of networks-multiple global ISPs

    • 两层结构：全球承载供应商（顶层）、接入ISP（底层）

  • 网络结构3：多层等级结构，Internet structure 3/4：network of networks-regional ISP+tier-1 ISP

  • 网络结构4：（在3的基础上）由接入ISP、区域ISP、第一层ISP、PoP、多宿、对等、IXP组成的生态系统

    • 存在点（PoP，Point of Presence）
      • 存在于等级结构的所有层次，底层（接入ISP）除外
      • 是提供商网络中的一台或多台路由器（在相同位置）群组，其中客户ISP能与提供商ISP连接
      • 对于要与提供商PoP连接的客户网络，它能从第三方通信提供商租用高速链路直接将它的路由器之一连接到位于该PoP的一台路由器
    • 多宿（multi-home）
      • **任何ISP（除了第一层ISP）**可以与两个或更多提供商ISP连接
      • 当一个ISP多宿时，即使它的提供商之一出现故障，它仍然能够继续**发送和接收分组*
    • 对等（peer）
      • 位于相同等级结构层次的邻近一对ISP能够对等，即能够直接将它们的网络连接到一起，使它们之间的所有流量直接连接而不是通过上游的中间ISP传输
    • 因特网交换点（IXP，Internet Exchange Point）
      • 通常在一个有自己的交换机的独立建筑物中
      • 是一个汇合点，多个ISP能够在这里共同对等

  • 网络结构5：在4的顶部增加内容提供商网络（content provider network）

    network of networks-Content provider network

Intern vs Intranet

  • loosely hierarchical (松散分层)

    • Both use e-mail and World Wide Web standards

  • What’s the differences？

    • An Intranet is local and private
    • An Intranet is more secure and more speed
    • An Intranet is regulated(管制)
