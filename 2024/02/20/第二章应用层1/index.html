第二章-应用层（一）

2.1 Principles of Network Applications

  • network-core devices do not function at the application laver but instead function at lower lavers—specifically at the network layer and below

==》confining application software to the end systems

  • network -core devices do not run user applications

2.1.1 Network Application Architectures，网络应用程序体系结构

  • From the application developer’s perspective, the network architecture is fixed and provides a specific set of services to application

  • application architecture：is designed by the application developer and dictates(规定，指定) how the application is structured over the various end systems

  • two predominant(主导的) architectural paradigms used in modern network applications:

    • the client-server（客户-服务器） architecture
    • the peer-to-peer (P2P，对等) architecture

  • client-server（客户-服务器） architecture

    • server

    • an always-on host

    • rvices requests from many other hosts, called clients

    • client

      • sometimes-on or always-on
      • intermittently connected (间断地连接)
      • may have dynamic(动态的) IP addresses

    • clients do not directly communicate with each othe

    • the server has a fixed, wellknown adress, called an IP address

    • Some of the applications with a client-server architecture include the Web, FTP, Telnet, and e-mail

    • a single server host is incapable of keeping up with all the requests from its clients

      ==》a large cluster of hosts（a data center）is often used to create a powerful virtual server in client-server architectures

    • Application services（应用服务程序） that are based on the client-server architecture are often infrastructure intensive

  • P2P architecture，P2P体系结构

    • there is minimal (or no) reliance on always-on infrastructure servers

    • peers：the application exploits direct communication between pairs of intermittently(间歇地) connected hosts

    • The peers(对等方) are not owned by the service provider，but are instead desktops and laptops controlled by users

    • 新型流量密集型应用，e.g. IM(Instant Message),IPTV(IP网络电视), Internet Phone, Live Streaming

      • BitTorrent, eMule, Skpye, PPTV, Thunder

    • the peers communicate without passing through a dedicated server（peer-to-peer）

    • some applications have hybrid architectures

      • many instant messaging applications：servers are used to track the IP addresses of users, but user-to-user messages are sent directly between user hosts (without passing through intermediate servers)，对于许多即时讯息应用而言，服务器被用于跟踪用户的IP地址，但用户到用户的报文在用户主机之间（无需通过中间服务器）直接发送

    • compelling features：self-scalability（自扩展性，可收缩性）

      • For example, in a P2P file-sharing application,although each peer generates workload by requesting files, each peer also adds service capacity to the system by distributing files to other peers，在一个P2P文件共享应用中，尽管每个对等方都由于请求文件产生工作量，但每个对等方通过向其他对等方分发文件也为系统增加服务能力
      • new peers bring new service capacity, as well as new service demands

    • cost effective

      • they normally don’t require significant server infrastructure and server bandwidth

    • major challenges：

    • ISP Friendly

      • most residential ISPs(including DSL and cable ISPs) have been dimensioned for “asymmetrical” bandwidth usage
      • P2P video streaming and file distribution applications shift upstream traffic from servers to residential ISPs, thereby putting significant stress on the ISPs

    • security

      • highly distributed and open nature，高度分布和开放特性

    • Incentives

      • The success of future P2P applications also depends on convincing users to volunteer bandwidth, storage, and computation resources to the applications

  • hybrid of client-server and P2P

    • Instant messaging applications
      • centralized service: client presence detection/location
        • user registers its IP address with central server when it comes online
        • user contacts central server to find IP addresses of buddies
      • chatting between two users is P2P
    • Skype(网络电话)
      • voice-over-IP (VoIP) P2P application
      • centralized server: finding address of remote party
      • client-client connection: direct (not through server)

2.1.2 Processes Communicating

  • In the jargon of operating systems,it is not actually programs but processes that communicate
  • when processes are running on the same end system, they can communicate with each other with inter-process communication（IPC，进程间通信）, using rules that are governed by the end system’s operating system
  • :star:processes on two different end systems communicate with each other by exchanging messages（报文） across the computer network
    • sending process: creates and sends messages into the network
    • receiving process: receives these messages and possibly responds by sending messages back
1.Client and Server Processes

  • A network application consists of pairs of processes that send messages to each other over a network（label one of the two processes as the client and the other process as the server)

    • in the Web application a client browser process exchanges messages with a Web server process
    • In a P2P file-sharing system,a file is transferred from a process in one peer to a process in another peer

  • define the client and server processes as follows:

    In the context of a communication session between a pair of processes, the process that initiates the communication(that is, initially contacts the other process at the beginning of the session)is labeled as the client.The process that waits to be contacted to begin the session is the server

  • sometimes also use the terminology “client side and server side of an application”

2.The Interface Between the Process and the Computer Network，进程与计算机网络之间的接口

  • Any message sent from one process to another must go through the underlying network

  • A process sends messages into, and receives messages from, the network through a software interface called a socket（套接字）

  • a socket is the interface between the application layer and the transport layer within a host

  • It is also referred to as the Application Programming Interface(API，应用程序编程接口） between the application and the network, since the socket is the programming interface with which network applications are built（建立网络应用程序的可编程接口）

  • The only control that the application developer has on the transport-layer side is

    • the choice of transport protocol
    • perhaps the ability to fix a few transport-layer parameters，such as maximum buffer and maximum segment sizes最大缓存和最大报文段长度
3、进程寻址
  • 主机由其IP地址（IP address）标识
  • to receive messages, process must have identifier
  • an IP address is a 32-bit quantity（uniquely identifying the host）
  • 除了知道报文送往目的地的主机地址外，发送进程还必须指定运行在接收主机上的接收进程（更具体地说，接收套接字），端口号（port number）用于此
App-layer protocol defines
  • types of messages exchanged
    • request, response
  • message syntax (语法)
    • what fields in messages & how fields are delineated(描绘)
  • message semantics (语义)
    • meaning of information in fields
  • rules for when and how processes send & respond to messages
  • open protocols (公开协议)
    • defined in RFCs
    • allows for interoperability( 互通性)
    • HTTP, SMTP
  • proprietary protocols (私有协议)
    • Skype, QQ, Thunder

2.1.3 Transport Services Available to Applications，可供应用程序使用的运输服务

  • 在发送端的应用程序将报文推进该套接字，在该套接字的另一侧，运输层协议负责使该报文进入接收进程的套接字
  • the possible services that a transport-layer protocol can offer：
    • reliable data transfer，可靠数据传输
    • throughput
    • timing，定时
    • security
reliable data transfer
  • data loss can have devastating consequences（灾难性的后果）
    • electronic mail
    • file transfer
    • remote host access，远程主机访问
    • Web document transfers
    • financial applications，金融应用
  • reliable data transfer：the data sent by one end of the application is delivered correctly and completely to the other end of the application，应用程序的一端发送的数据正确、完全地交付给该应用程序的另一端
  • One important service that a transport-layer protocol can potentially provide to an application is process-to-process reliable data transfer（可靠数据传输）
  • loss-tolerant applications（容忍丢失的应用）：
  • real-time audio/video
  • stored audio/video
Throughput
  • throughput：in the conext of a communication session between two processes along a network path is the rate at which the sending process can deliver bits to the receiving proces，在沿着一条网络路径上的两个进程之间的通信会话场景中，可用吞吐量就是发送进程能够向接收进程交付比特的速率
  • the available throughput can fluctuate with time
  • transport-layer protocol could provide another natural service：guaranteed available throughput at some specified rate
  • With such a service,，the application could request a guaranteed throughput of r bits/sec，and the transport protocol would then ensure that the available throughput is always at least r bits/sec
  • bandwidth-sensitive applications（宽带敏感的应用）：Applications that have throughput requirements
    • Many current multimedia applications are bandwidth sensitive, although some multimedia applications may use adaptive coding techniques to encode at a rate that matches the currently available throughpu
  • elastic applications（弹性应用）：can make use of as much,or as little, throughput as happens to be available
    • Electronic mail
    • file transfer
    • Web transfers，web传送
Timing（定时）
  • A transport-layer protocol can also provide timing guarantees
  • Internet telephony、virtual environments、teleconferencing、multiplayer games
  • For non-real-time applications, lower delay is alwavs preferable to higher delay, but no tight constraint is placed on the end-to-end delays
Security
  • a transport protocol can provide an application with one or more security services
  • in the sending host, a transport protocol can encrypt all data transmitted by the sending process,and in the receiving host, the transport-layer protocol can decrypt the data before delivering the data to the receiving process

TCP Services
  • TCP service=connection-oriented service + reliable data transfer service，TCP服务模型=面向连接服务+可靠数据传输服务
  • connection-oriented service
    • before the application-level messages begin to flow，TCP has the client and server exchange transport-layer control information with each other （handshaking procedure：prepare for an onslaught of packets）
    • After the handshaking phase，a TCP connection is said to exist between the sockets of the two processes
    • The connection is a full-duplex connection（全双工）：the two processes can send messages to each other over the connection at the same time
    • When the application finishes sending messages, it must tear down the connection
    • the two processes are connected in a very loose manner
  • reliable data transfer service
    • deliver all data sent without error and in the proper order，无差错、按适当顺序交付所有发送的数据
    • no missing or duplicate bytes，没有字节的丢失和冗余
  • TCP also includes a congestion-control mechanism（拥塞控制机制）（a service for the general welfare of the Internet rather than for the direct benefit of the communicating processes）
    • The TCP congestion-control mechanism throttles a sending process(client or server) when the network is congested between sender and receiver
    • TCP congestion control also attempts to limit each TCP connection to its fair share of network bandwidth，TCP拥塞控制也试图限制每个TCP连接、使它们达到公平共享网络带宽的目的
  • summary
    • reliable transport可靠传输、flow control流量控制、congestion control拥塞控制、connection-oriented面向连接
    • no：
      • timing、minimum throughput guarantee(最低保证吞吐量)、security
UDP Services
  • no-frills、lightweight、providing minimal services、connectionless（=》no handshaking）
  • provides an unreliable data transfer service
    • no guarantee that the message will ever reach the receiving process. Furthermore, messages that do arrive at the receiving process may arrive out of order，乱序
  • does not include a congestion-control mechanism
    • the sending side of UDP can pump data into the layer below(the network layer) at any rate it pleases
    • the actual end-to-end throughput may be less than this rate due to the limited bandwidth of intervening links or due to congestion
  • summary
    • does not provide: reliability，flow control，congestion control，timing，throughput guarantee，security，connection setup
focus on security

  • Neither TCP nor UDP provide any encryption

  • Secure Sockets Layer(SSL，安全套接字层）：an enhancement for TCP

  • not only···（TCP），also provides critical process-to-process security services

    • encryption
    • data integrity，数据完整
    • end-point authentication，端点鉴别

  • :star:SSL is not a third Internet transport protocol,on the same level as TCP and UDP, but instead is an enhancement of TCP，with the enhancements being implemented in the application layer，SSL 不是与TCP和 UDP在相同层次上的第三种因特网 运输协议，而是一种对TCP的加强，这种强化是在应用层上实现的

  • if an application wants to use the services of SSL, it needs to include SSL code in both the client and server sides of the applicaion

  • SSL has is own socket API that is similar to the traditional TCP socket API

  • about throughput or timing guarantees（services not provided by today’s Internet transport protocol）（in ch.7）

    • today’s Internet can often provide satisfactory service to time-sensitive applications, but it cannot provide any timing or bandwidth guarantees，今天的因特网通常能够为时间敏感应用提供满意的 服务，但它不能提供任何定时或带宽保证

  • 一些流行的因特网应用所使用的运输协议：

    • telephony typically runs over UDP
      • Each side of an Internet phone application needs to send data across the network at some minimum rate.this is more likely to be possible with UDP than with TCP
      • Also,Internet phone applications are loss-tolerant, so they do not need the reliable data transfer service provided by TCP.因为因特网电话应用（如Skype）通常能够容忍某些丢失但要求达到一定的最小速率才能有效工作，所以常将该应用运行在 UDP上，从而设法避开 TCP 的拥塞控制机制和分组开销
      • 但因为许多防火墙被配置成阻挡（大多数类型的）UDP流量，所以因特网电话应用通常设计成如果 UDP通信失败就使用 TCP 作为备份

2.1.5 Application-Layer Protocols

  • An application-layer protocol defines how an application’s processes, running on different end systems, pass messages to each other：
    • The types of messages exchanged，交换的报文类型
      • for example, request messages and response messages
    • The syntax of the various message types，各种报文类型的语法
      • such as the fields in the message and how the fields are delineated，报文中的各个字段及这些字段是如何描述的
    • The semantics of the fields，字段的语义
      • the meaning of the information in the fields
    • Rules for determining when and how a process sends messages and responds to messages，一个进程何时以及如何发送报文，对报文进行响应的规则
  • Some application-layer protocols are specified in RFCs and are therefore in the public domain
    • For example, the Web’s application-layer protocol, HTTP(the HyperText Transfer Protocol is available as an RFC
  • Many other application-layer protocols are proprietary（专用的）
    • many existing P2P file-sharing systems use proprietary application-layer protocols：Skype
