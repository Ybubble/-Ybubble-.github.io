3.6 Principles of Congestion Control，塞控制原理

congestion control in the available bit-rate(ABR，可用比特率)service in asynchronous transfer mode(ATM，异步传递方式) networks，异步传递方式（ATM）网络中可用比特率（ABR）服务中的拥塞控制

  • congestion:
    • informally: “too many sources sending too much data too fast for network to handle”
    • different from flow control! (receiver controls sender, so sender won’t overflow receiver’s buffer) (注意与流量控制的区别！)
    • Manifestations(表现):
      • long delays (queueing in router buffers)
      • lost packets (buffer overflow at routers)
    • a top-10 problem!

3.6.1 The Causes and the Costs of Congestion

Scenario 1: Two Senders, a Router with Infinite Buffers，两个发送方和一台具有无穷大缓存的路由器

  • output link capacity: R

    no retransmission, flow control, and congestion control

  • 每连接的吞吐量（per-connection throughput）（接收方每秒接收的字节数）与该连接发送速率之间的函数关系

    • The link simply cannot deliver packets to a receiver at a steady-state rate that exceeds R/2，链路完全不能以超过 R/2的稳定状态速率向接收方交付分组
    • This upper limit on throughput is a consequence of the sharing of link capacity between two connections
    • No matter how high Hosts A and B set their sending rates, they will each never see a throughput higher than R/2

  • When the sending rate exceeds R/2, the average number of queued packets in the router is unbounded(无限的，极大的，不受控的), and the average delay between source and destination becomes infinite

  • 拥塞网络的一种代价：large queuing delays are experienced as the packet arrival rate nears the link capacity

Scenario 2: Two Senders and a Router with Finite Buffers, 两个发送方和一台具有有限缓存的路由器

  • one router, finite buffers (likely droped packet)

  • sender retransmission of timed-out packet

  • λ~in~ :Sends original data into socket

  • λ’~in~: Trsnsport layer sends segments into network（containing ontaining original data and retransmitted data，sometimes referred to as the offered load，供给载荷）

  • application-layer input = application-layer output: λ~in~ = λ~out~

  • transport-layer input includes retransmissions :λ‘~in~ ≥λ~in~

①Idealization: perfect knowledge，理想情况：A能确定路由器中的缓存是否空闲，因而仅当缓存空闲时才发送一个分组。在这种情况下，将不会产生丢包，λ~in~与λ’~in~相等，并且连接的吞吐量就等于λ~in~

平均主机发送速率不能超过 R/2，因为假定不会发生分组丢失

从吞吐量的角度看、性能理想，即发送的每个分组都被接收到

② Idealization: known loss packets can be lost, dropped at router due to full buffers，发送方仅当在确定了一个分组已经丢失时才重传

因此，在所发送的 0.5R 单位数据当中，从平均的角度说， 0.333R 字节/秒是初始数据，而0.166R字节/秒是重传数据。

the sender must perform retransmissions in order to compensate for dropped (lost) packets due to buffer overflow另一种网络拥寒的代价，即发送方必须执行重传以补偿因为缓存溢出而丢弃（丢失）的分组

③Realistic: duplicates

  • packets can be lost, dropped at router due to full buffers
  • sender times out prematurely, sending two copies, both of which are delivered
  • 发送方也许会提前发生超时并重传在队列中已被推迟但还未丢失的分组
  • 初始数据分组和重传分组都可能到达接收方（接收方只需要一份这样的分组副本就行了，重传分组将被丢弃）
  • 在这种情况下，路由器转发重传的初始分组副本是在做无用功，因为接收方已收到了该分组的初始版本。而路由器本可以利用链路的传输能力去发送另一个分组
  • unneeded retransmissions by the sender in the face of large delays may cause a router to use its link bandwidth to forward unneeded copies of a packet，网络拥塞的另一种代价，即发送方在遇到大时延时所进行的不必要重传会引起路由器利用其链路带宽来转发必要的分组副本
  • “costs” of congestion:
    • more work (retrans) for given “goodput”(实际吞吐量)
    • unneeded retransmissions: link carries multiple copies of pkt decreasing goodput
Scenario 3: Four Senders, Routers with Finite Buffers, and Multihop Paths，4个发送方和具有有限缓存的多台路由器及多跳路径

  • assume that each host uses a timeout/retransmission mechanism to implement a reliable data transfer service，都采用超时/重传机制来实现可靠数据传输服务

    all hosts have the same value of λ~in~

    all router links have capacity R bytes/sec

    all the router have R capacity

  • 对于极小的 λ~in~值，路由器缓存的溢出是很少见的，吞吐量大致接近供给载荷，对稍大的 λ~in~值，对应的吞吐量也更大，因为有更多的初始数据被发送到网络中并交付到目的地，溢出仍然很少。因此，对于较小的λ~in~，λ~in~的增大会导致λ~out~的增大

  • λ~in~（因此λ’~in~）很大时的情况:

    • 到达路由器 R2的A-C流量（在经过路由器 R1转发后到达路由器R2）的到达速率至多是R，也就是从 R1 到 R2 的链路容量

    • 如果λ’~in~对于所有连接 （包括B-D连接）来说是极大的值，那么在 R2上，B-D流量的到达速率可能会比A-C 流量的到达速率大得多，存在竞争

    • as red λ‘~in~ increases, all arriving blue pkts at upper queue are dropped, blue throughput->0

    • the transmission capacity used at the first router to forward the packet to the second router could have been much more profitably used to transmit a different packet. (For example, when selecting a packet for transmission, it might be better for a router to give priority to packets that have already traversed some number of upstream routers.)

  • another cost of dropping a packet due to congestion—when a packet is dropped along a path, the ransmission capacity that was used at each of the upstream links to forward that packet to the point at which it is dropped ends up having been wasted
  • another “cost” of congestion:
    • when packet dropped, any upstream transmission capacity (routers) used for that packet was wasted!
    • 当一个segment沿着一条路径传输被丢弃时，上游路由器转发该分组的传输容量就被浪费了

3.6.2 Approaches to Congestion Control

two broad approaches to congestion control that are taken in practice（ distinguish among congestion-control approaches by whether the network layer provides any explicit assistance to the transport layer for congestion-control purposes）

  • end-to-end congestion control，端到端拥塞控制
    • the network layer provides no explicit support to the transport layer for congestion control purposes，没有显式支持
    • Even the presence of congestion in the network must be inferred by the end systems based only on observed network behavior(for example, packet loss and delay)
    • TCP must necessarily take this end-to-end approach toward congestion control
      • since the IP layer provides no feedback to the end systems regarding(有关) network congestion
      • TCP segment loss(as indicated by a timeout or a triple(3次) duplicate acknowledgment) is taken as an indication of network congestion and TCP decreases its window size accordingly(相应地，因而)
      • uses increasing round-trip delay values as indicators of increased network congestion，使用增加的往返时延值作为网络拥塞程度增加的指示
  • Network-assisted congestion control，网络辅助的拥塞控制
    • network-layer components(网络层构件) (that is, routers)provide explicit feedback to the sender regarding the congestion state in the network，提供关于网络中拥塞状态的显式反馈信息. This feedback may be as simple as a single bit indicating congestion at a line
    • congestion information is typically fed back from the network to the sender in one of two ways
      • derect feedback may be sent from a network router to the sender. This form of notification typically takes the form of a choke packet(阻塞分组) (essentially saying,“I’m congested!”)
    • the second form of notification occurs when a router marks/updates a field in a packet flowing from sender to receiver to indicate congestion
      • Upon receipt of a marked packet, the receiver then notifies the sender of the congestion indication
      • Note that this latter form of notification takes at least a full round-trip time

3.6.3 Network-Assisted Congestion-Control Example: ATM ABR Congestion Control, 网络辅助的拥塞控制例子∶ATM ABR拥塞控制

  • Fundamentally(基本上) ATM takes a virtual-circuit (VC) oriented approach toward packet switching，采用一种面向虚电路（VC）的方法来处理分组交换

  • each switch on the source-to-destination path will maintain state about the source-to-destination VC

  • This per-VC state allows a switch to track the behavior of individual senders (e.g.,tracking their average transmission rate) and to take source-specific congestion-control actions(采取特定源的拥塞控制动作) (such as explicitly signaling to the sender to reduce its rate when the switch becomes congested)

  • This per-VC state at network switches makes ATM ideally suited to perform network-assisted congestion control

  • ABR has been designed as an elastic(弹性的) data transfer service in a manner reminiscent(使人联想) of TCP

    • When the network is underloaded, ABR service should be able to take advantage of the spare available bandwidth
    • when the network is congested, ABR service should throttle its transmission rate to some predetermined minimum transmission rate

  • adopt ATM terminology(using the term switch(交换机) rather than router, and the term cell(信元) rather than packet)

  • Interspersed(穿插) with the data cells are resource-management cells (RM cells)(资源管理信元，RM信元)

    • can be used to convey congestion-related information among the hosts and switches

    • When an RM cell arrives at a destination, it will be turned around and sent back to the sender(possibly after the destination has modified the contents of the RM cell)

    • It is also possible for a switch to generate an RM cell itself and send this RM cell directly to a source

    • RM cells can be used to provide both direct network feedback and network feedback via the receiver(提供直接网络反馈和经由接收方的网络反馈)

  • ATM ABR congestion control is a rate-based(基于速率) approach

    • the sender explicitly computes a maximum rate at which it can send and regulates(调整，控制) itself accordingly，明确计算最大速率，自我调整

  • ABR provides three mechanisms for signaling congestion-related information from the switches to the receiver:

    • EFCl bit
      • Each data cell contains an explicit forward congestion indication (EFCI，式转发拥塞指示) bit
      • (congested network switch can set)置1：告诉目的主机已经拥塞
      • The destination must check the EFCI bit in all received data cells
      • When an RM cell arrives at the destination, if the most recently received data cell had the EFCI bit set to 1, then the destination sets the congestion indication bit (the CI bit，拥塞指示比特) of the RM cell to 1 and sends the RM cell back to the sender
    • CI and NI bits
      • sender-to-receiver RM cells are interspersed with data cells. The rate of RM cell interspersion is a tunable(可调的) parameter(参数), with the default value being one RM cell every 32 data cells，每32个数据信元中有一个RM信元
      • These RM cells have a congestion indication(CI，拥塞指示) bit and a no increase (NI，无增长) bit that can be set by a congested network switch，拥塞的交换机设置
      • switch can set the NI bit in a passing RM cell to 1 under mild congestion and can set the CI bit to 1 under severe congestion conditions
      • When a destination host receives an RM cell, it will send the RM cell back to the sender with its CI and NI bits intact(不变的) (除了上述EFCI的情况)
    • ER setting
      • Each RM cell also contains a 2-byte **explicit rate (ER，显式速率)**field
      • A congested switch may lower the value contained in the ER field in a passing RM cell
      • the ER field will be set to the minimum supportable rate of all switches on the source-to-destination path
