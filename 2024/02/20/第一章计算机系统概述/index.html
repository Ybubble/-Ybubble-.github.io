第一章-操作系统概述

操作系统各主题间的关联性

基本构成

处理器：

  • processor

  • 控制计算机的操作，执行数据处理功能

    • 与存储器交换数据

    • 通常使用两个内部（对处理器而言）寄存器：

      • 存储器地址寄存器，Memory Address Register，MAR

        用于确定下一次读/写的存储器地址

      • 存储器缓冲寄存器，Memory Buffer Register，MBR

        存放要写入存储器的数据或从存储器中读取的数据

    • 输入/输出地址寄存器，I/O Address Register，I/O AR，I/O地址寄存器

      用于确定一个特定的输入/输出设备

    • 输入/输出缓冲寄存器，I/O Buffer Register，I/O BR，I/O缓冲寄存器

      用于在输入/输出模块和处理器间交换数据

  • 只有一个处理器时，通常指中央处理器（CPU）

内存：
  • main memory
  • 存储数据和程序
  • 存储易失性，具有非易失性的是磁盘存储器
  • 亦称为实存储器real memory、主存储器primary memory
  • 内存模块由一组单元组成，这些单元由顺序编号的地址定义
  • 每个单元包含一个二进制数，可以解释为一个指令或数据
输入/输出模块：

  • I/O modules

  • 在 计算机和外部环境之间移动数据

  • 外部环境由各种外部设备组成：

    • 辅助存储设备（硬盘等）
    • 通信设备
    • 终端

  • 包含内存缓冲区

    保存临时文件，直至被发送出去

数据总线：
  • 在处理器、内存、输入/输出设备之间提供通信的设施

微处理器的发展

  • 微处理器可以在单个芯片上容纳一个处理器

  • 处理速度：慢······（硬件上的交互信息缩短到纳秒量级）······>快

  • 如今微处理器成为了最快的通用处理器

  • 发展成为了多处理器：

    • 每个芯片（底座）上容纳了多个处理器（内核）

    • 每个处理器上有多层大容量缓存

    • 多个处理器之间共享内核的执行单元

  • 数字信号处理器

    对流信号（如音频、视频）进行处理

  • 其他专门的计算装置（固定功能单元）与CPU共存，用来支持其他标准的一些计算，如：

    • 编码/解码语音和视频（多媒体数字信号编码器/解码器）
    • 提供对加密和安全技术的支持

  • 传统微处理器正在被片上系统（便携）所取代:

    CPU和高速缓存在一个芯片上

    系统中的多数其他硬件也在此芯片上，如数字信号处理器、图像处理单元、I/O装置（无线电和多媒体数字信号编码器/解码器）、内存

指令的执行

  • 一组（保存在存储器中的）指令—>（处理器执行的）程序

    （不断重复）取指令+执行指令—>程序执行

  • 最简单的指令处理：

    1、处理器从存储器中一次读取一条指令

    2、执行每条指令

  • 指令周期

    • 单个指令所需要的处理
    • 取指阶段 + 执行阶段
    • 仅当机器关机、发生某些未知错误、遇到与停机相关的程序指令时，程序执行才会停止
    • 程序计数寄存器
      • Program Counter，PC
      • 保存下一次要取的地址
      • 若非出现其他情况，处理器会在每次取指后递增PC，以便能按顺序取下一条指令（即位于下一个存储器地址的指令）
    • 指令寄存器
      • Instruction Register，IR
      • 存放取到的指令（指令中包含处理器将要执行的操作的位，处理器解释指令并执行相应的操作）

  • 例：累加，把地址为940的内存单元中的内容与地址为941的内存单元中的内容相加，并将结果保存在后一个单元中

    AC：累加器（数据寄存器）

    程序计数器（PC）=指令的地址

    指令寄存器（IR）=正在被执行的指令

    累加器（AC）=临时存储

中断

分类
类型 定义
程序中断 在某些条件下由指令执行的结果产生，如算数溢出、除数为0、试图执行一条非法机器指令及访问用户不允许的存储器位置
时钟中断 由处理器内部的计时器产生，允许操作系统以一定的规律执行函数
I/O中断 由I/O控制器产生，用于发信号通知一个操作的正常完成或各种错误条件
硬件失效中断 由诸如掉电或存储器奇偶校验错之类的故障产生
中断和指令周期

  • 例：WRITE调用

中断处理程序执行过程中，外部设备忙于从计算机存储接收数据并打印，这种I/O操作和用户程序中指令的执行是并发的

过程：1、外部设备做好服务的准备（准备好从处理器接收更多数据）

​ 2、外部设备的I/O模块给处理器发送一个中断请求信号

​ 3、处理器响应：

​ 暂停当前程序的处理

​ 处理服务于特定I/O设备的程序（中断处理程序，interrupt handler）

​ 4、完成对设备的服务响应后，处理器恢复原先的执行

  • 中断可在主程序中的任何位置而非仅在一条特定指令处发生

  • 处理器检查是否出现中断信号：

    • 无中断：

      处理器继续运行，并在取值周期取当前程序的下一条指令

    • 有中断：

      处理器挂起当前程序的执行，并执行一个中断处理程序（通常是操作系统的一部分，确定中断的性质并执行所需要的操作）

中断处理

  • PSW：程序状态字

  • PC：程序计数器

  • 控制权交到中断处理程序后，该程序执行会引起以下操作：

    • 系统栈中的信息：

      • 与被中断相关的程序计数器
      • PSW
      • 一些其他信息被当作正在执行的程序中的一部分
      • 与处理寄存器相关的内容（中断处理程序可能用到这些寄存器）

    • 中断处理程序开始处理中断：

      • 检查与I/O操作相关的状态信息或其他引起中断的事件
      • 可能包括给I/O设备发送附加命令或应答

    • 中断处理结束后：

      被保存的寄存器值从栈中释放并恢复到寄存器中

    • 从栈中恢复PSW和程序计数器的值

      （因此下一条要执行的指令来自前面被中断的程序）

多个中断

处理多个中断的方法：

  • 正在处理一个中断时，禁止再发生中断：

    • 处理器检查是否还有中断发生

    • 所有中断都严格按照顺序处理

    • 缺点：未考虑相对优先级和时间限制的要求

      例如，当来自通信线的输入到达时可能需要快速接收，以便为更多的输入让出空间。如果在第二批输入到达时第一批输入还未处理完，就可能由于I/O设备的缓冲区装满或溢出而丢失数据

  • 定义中断优先级，允许高优先级中断打断低优先级中断

存储器的层次结构

  • 存储器评价：
    • 容量：供应用程序使用

    • 存取事件快，每“位”价格高
    • 容量大，每“位”价格低
    • 容量大，存取速度慢

高速缓存

动机

就目前而言，处理器速度的提高一直快于存储器访问速度的提高。理想情况下，内存的构造技术可采用与处理器中的寄存器相同的构造技术，这样内存的存储周期才跟得上处理周期（但成本高）

解决办法：利用构造性原理（principal of locality），即在内存之间提供一个容量小且速度快的存储器，称为高速缓存

高速缓存原理

  • 内存：容量相对较大但速度较慢

    高速缓存：容量较小但速度较快，包含一部分内存数据的副本

  • 处理器读取存储器中的一个字节或字之前：检查—>确定该字或字节是否在高速缓存中

    • 在：

      该字节/字从高速缓存传递给处理器

    • 不在：

      1、将由固定数目的字节/字组成的一块内存数据读入高速内存

      2、该字节/字从高速内存传递给处理器

  • 根据访问局部性原理，当一块数据被取入高速缓存以满足一次存储器访问时，很可能紧接着的多次访问的数据是该块中的其他字节

执行I/O操作的技术
  • 可编程I/O
  • 中断驱动I/O
  • 直接内存存取（DMA）
可编程I/O操作（programmed I/O）
  • I/O模块执行请求的动作并设置I/O状态寄存器中相应的位
  • 不会进一步通知处理器
  • 不会中断处理器
  • 处理器在执行I/O指令后，要定期检查I/O模块的状态，以确定I/O操作是否完成
  • 缺点：
    • 处理器要等很长时间来缺点I/O模块是否做好了接收或发送更多数据的准备
    • 处理器在等待期间必须不断询问I/O模块的状态，会降低整个系统的性能
    • I/O传送速度受限于处理器测试设备和提供服务的速度
    • 处理器忙于管理I/O传送的工作，必须执行很多指令以完成I/O传送
中断驱动I/O（interrupt-driven I/O）
  • 处理器给I/O模块发送I/O命令，然后处理器忙其他有用的事情
  • I/O模块准备好与处理器交换数据时，打断处理器的执行并请求服务
  • 缺点：
    • 任何数据传送都必须完全通过处理器
    • I/O传送速度受限于处理器测试设备和提供服务的速度
    • 处理器忙于管理I/O传送的工作，必须执行很多指令以完成I/O传送
直接内存存取（Direct Memory Access，DMA）
  • 需要大量移动数据时更有效
  • DMA功能可以由系统总线中的一个独立模块完成，也可以并入一个I/O模块中
  • 过程：
    • 处理器读或写一块数据时，给DMA模块产生一条命令，发送：
      • 是否请求一次读或写
      • 所涉I/O设备的地址
      • 开始读或写的存储单元
      • 需要读或写的字数
    • 处理器继续其他工作
    • DMA模块直接与存储器交互，发送整个数据块，每次传送一个字
    • 传送完成后，DMA模块向处理器发送一个中断信号
  • 只有在开始传送和传送结束时处理器才会参与
  • DMA需要控制总线来与存储器进行数据传送
  • 由于在总线使用中存在竞争，当处理器需要使用总线时，要等待DMA模块（并非中断，处理器没有保存上下文环境去做其他事，而只是暂停一个总线周期[在总线上传输一个字的时间]）
